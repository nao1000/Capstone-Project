Namespace: localSchedule

Scheduler~localSchedule

An in-memory store for managing shift data before it is persisted to the backend database. This acts as a client-side cache to retain unsaved changes (like newly drawn shifts) while the user navigates between different views (e.g., switching role filters).
Properties:
Name Type Description
shifts Object.<string, Object> A dictionary storing shift objects, indexed by a composite string key (`${roleId}-${day}-${start_min}`).
